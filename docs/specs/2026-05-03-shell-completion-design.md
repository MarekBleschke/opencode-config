# Shell Autocompletion for oc-sandbox

**Date:** 2026-05-03  
**Type:** Technical Design  
**Status:** Draft  
**Supersedes:** Extends `2026-05-01-opencode-sandbox-design.md`

## Summary

Add shell autocompletion support to `oc-sandbox`, starting with Zsh. The implementation follows the standard `source <(tool completion <shell>)` pattern used by `kubectl`, `podman`, `helm`, and other CLI tools. Completions are installed by default with the `install` subcommand (opt-out with `--no-completions`) and cleaned up by `uninstall`. Dynamic completions for image tags and profiles use a 24h TTL file-based cache with background refresh, ensuring tab-completion is always instant.

## Goals

- **Zsh completions in v1** — Full subcommand, flag, and dynamic value completion for Zsh
- **Extensible architecture** — Bash and Fish can be added later by extending the `completion` subcommand
- **Instant tab-completion** — Dynamic values (image tags, profiles) are cached; tab press never calls podman synchronously
- **Automatic cache freshness** — `oc-sandbox build` auto-refreshes cache; stale cache triggers background refresh
- **Seamless install/uninstall** — Completions are set up by default during `install` and cleaned up during `uninstall`
- **Symlink-safe** — Works correctly when `.zshrc` is a symlink

## Non-Goals

- Bash and Fish completion implementations (architecture is ready, but not implemented in v1)
- Completion for dynamic values other than image tags and profiles (e.g., workspace paths — these are already handled by `_files -/`)
- GUI or TUI-based completion configuration
- Global completion registration via `/usr/share/zsh/site-functions/` (user-level only via `.zshrc`)

## Design Decisions

1. **Self-contained completion generation** — Completion scripts are generated by the `oc-sandbox` script itself via heredocs, not stored as separate files. This preserves the single-file deployment model and follows the pattern of tools like `kubectl`.
2. **`source <(oc-sandbox completion zsh)` pattern** — The standard, well-established pattern for shell completion registration. Works both when sourced in the current shell and when added to `.zshrc`.
3. **24h TTL file-based cache** — Cache files in `~/.cache/oc-sandbox/` with timestamps in `cache.meta`. On cache miss, stale data is returned and a background refresh is triggered. This ensures tab-completion is always instant.
4. **Per-image profile caching** — Profiles differ per image tag. Cache files are `profiles-<tag>`, so `oc-sandbox run --profile <TAB>` completes from the correct profile list based on the `--tag` context.
5. **Auto-refresh on build** — `oc-sandbox build` refreshes the cache for the built tag on success, so new images and profiles are immediately available for completion.
6. **Background refresh on stale cache** — When the completion function detects a stale cache (>24h), it spawns a background `oc-sandbox completion --refresh` process. The current tab press returns instantly from the stale cache; the next press gets fresh data.
7. **Cache seeding on install** — `oc-sandbox install` runs `oc-sandbox completion --refresh` to populate the initial cache.
8. **Symlink-safe RC file handling** — All `.zshrc` operations (append, grep, remove) naturally follow symlinks. An informational message is shown when `.zshrc` is a symlink.

## New `completion` Subcommand

### Command Structure

```
oc-sandbox completion <shell>        Generate completion script for <shell>
oc-sandbox completion --refresh     Force-refresh the completion cache
oc-sandbox completion --help        Show help message
```

### `oc-sandbox completion zsh`

Outputs a Zsh completion function. Usage:

```bash
source <(oc-sandbox completion zsh)
```

The generated function provides:

- **Subcommand completion** — `build`, `run`, `install`, `uninstall`, `completion`
- **Flag completion** — All flags for each subcommand (e.g., `--tag`, `--force`, `--profile`, `--no-ssh`, `--no-completions`)
- **Dynamic image tag completion** — `--tag` and `-t` values are completed from `~/.cache/oc-sandbox/images`
- **Context-aware profile completion** — `--profile` and `-p` values are completed from `~/.cache/oc-sandbox/profiles-<tag>`, where `<tag>` is determined by scanning the command line for a preceding `--tag`/`-t` argument (default: `main`)
- **Directory completion** — The PATH argument for `run` uses `_files -/`
- **Shell name completion** — The shell argument for `completion` completes `zsh` (and `bash`/`fish` with a "not implemented" message)

### `oc-sandbox completion --refresh`

Synchronously refreshes the completion cache:

1. Query `podman images --format '{{.Repository}}:{{.Tag}}'` and filter for `localhost/opencode-sandbox:*` to get available tags
2. Strip the `localhost/opencode-sandbox:` prefix to get tag names
3. Write tag names to `~/.cache/oc-sandbox/images`
4. For each tag, run `podman run --rm localhost/opencode-sandbox:<tag> ls -1 /home/sandbox/opencode-config/profiles/` to get available profiles
5. Write profile names to `~/.cache/oc-sandbox/profiles-<tag>`
6. Update `~/.cache/oc-sandbox/cache.meta` with current timestamps

If podman is not available, `--refresh` prints a warning and exits without error (cache remains empty).

### `oc-sandbox completion bash` / `oc-sandbox completion fish`

Prints a message: "Bash/Fish completion is not yet implemented. See 'oc-sandbox completion --help' for available shells." and exits 0.

### Help Text

```
Usage: oc-sandbox completion <shell>

Generate shell completion scripts for oc-sandbox.

Shells:
    zsh         Generate Zsh completion script

Options:
    --refresh           Force-refresh the completion cache
    --tag <TAG>         Refresh cache for a specific tag only (used with --refresh)
    -h, --help          Show this help message

Examples:
    source <(oc-sandbox completion zsh)    # Enable Zsh completions
    oc-sandbox completion --refresh        # Manually refresh all cached data
    oc-sandbox completion --refresh --tag main  # Refresh cache for tag "main" only
```

### Error Cases

| Condition | Handling |
|---|---|
| No shell argument | Error: "Shell name required. Use 'oc-sandbox completion --help' for available shells." |
| Unsupported shell (e.g., `completion bash`) | Info message: "Bash completion is not yet implemented." Exit 0 |
| Unknown shell (e.g., `completion ksh`) | Error: "Unknown shell 'ksh'. Available shells: zsh" |
| `--refresh` with no podman | Warning: "podman not found. Cache refresh skipped." Exit 0 |
| `--refresh` with no images | Write empty cache files. Exit 0 |

## Completion Cache

### Cache Location

`~/.cache/oc-sandbox/` — follows XDG Base Directory specification.

### Cache Files

| File | Content | Format |
|---|---|---|
| `images` | Available image tag names | One per line, e.g., `main`, `experiment` |
| `profiles-<tag>` | Available profiles for image `<tag>` | One per line, e.g., `dev`, `web-dev` |
| `cache.meta` | Timestamps for TTL tracking | Key=value pairs: `images_timestamp=1746288000`, `profiles-main_timestamp=1746288000` |

### Cache Behavior

| Scenario | Action |
|---|---|
| Cache hit (< 24h old) | Read from cache file. Instant. |
| Cache miss (stale or missing) | Read from cache file if it exists (even if stale). Trigger background refresh. If no cache file exists at all, return empty list and trigger background refresh. |
| `--refresh` flag | Synchronous refresh: query podman for images and profiles. Write to cache. |
| `oc-sandbox build` success | Auto-refresh cache for the built tag. |
| `oc-sandbox install` | Seed initial cache via `--refresh`. |
| `oc-sandbox uninstall` | Delete entire cache directory. |

### Background Refresh Mechanism

When the completion function detects a stale cache (>24h), it appends a background process invocation to the completion output:

```zsh
(oc-sandbox completion --refresh >/dev/null 2>&1 &)
```

This runs as a detached background process. It does not slow down the current tab press. The next tab press will get fresh data.

### Cache Directory Creation

The `cmd_completion` function creates `~/.cache/oc-sandbox/` on demand if it doesn't exist. No special initialization step is required.

## Zsh Completion Function

### Generated Function Structure

The `oc-sandbox completion zsh` command outputs a self-contained Zsh completion definition:

```zsh
#compdef oc-sandbox

_oc-sandbox() {
  local -a commands
  commands=(
    'build:Build a container image'
    'run:Run opencode in a sandbox'
    'install:Install oc-sandbox as a global command'
    'uninstall:Uninstall the global oc-sandbox command'
    'completion:Generate shell completion scripts'
  )

  if (( CURRENT == 2 )); then
    _describe 'command' commands
    _arguments \
      '--help' '-h[Show help message]'
    return
  fi

  case "$words[2]" in
    build)       _oc-sandbox_build ;;
    run)         _oc-sandbox_run ;;
    install)     _oc-sandbox_install ;;
    uninstall)   _oc-sandbox_uninstall ;;
    completion)  _oc-sandbox_completion ;;
  esac
}

_oc-sandbox_build() {
  _arguments \
    '(-t --tag)'{-t,--tag}'[Tag for the built image]:tag:_oc-sandbox_image_tags' \
    '(-f --force)'{-f,--force}'[Force rebuild even if image exists]' \
    '--help[Show help message]' \
    '(-h)'-h'[Show help message]'
}

_oc-sandbox_run() {
  _arguments \
    '(-t --tag)'{-t,--tag}'[Tag of the image to run]:tag:_oc-sandbox_image_tags' \
    '(-p --profile)'{-p,--profile}'[Opencode profile to activate]:profile:_oc-sandbox_profiles' \
    '--no-ssh[Skip mounting SSH keys]' \
    '--no-auth[Skip mounting auth.json]' \
    '--no-gh-token[Skip GH_TOKEN detection]' \
    '--gh-token[GitHub token]:token:' \
    '--debug[Run /bin/bash instead of opencode]' \
    '--help[Show help message]' \
    '(-h)'-h'[Show help message]' \
    ':path:_files -/'
}

_oc-sandbox_install() {
  _arguments \
    '--no-completions[Skip shell completion setup]' \
    '--help[Show help message]' \
    '(-h)'-h'[Show help message]'
}

_oc-sandbox_uninstall() {
  _arguments \
    '--help[Show help message]' \
    '(-h)'-h'[Show help message]'
}

_oc-sandbox_completion() {
  _arguments \
    ':shell:_oc-sandbox_shells' \
    '--refresh[Force-refresh the completion cache]' \
    '--help[Show help message]' \
    '(-h)'-h'[Show help message]'
}

_oc-sandbox_image_tags() {
  local cache_file="$HOME/.cache/oc-sandbox/images"
  if [[ -f "$cache_file" ]]; then
    _values 'image tags' $(cat "$cache_file")
  fi
  _oc-sandbox_maybe_refresh_cache
}

_oc-sandbox_profiles() {
  local tag="main"
  local i
  for (( i = 2; i < CURRENT; i++ )); do
    if [[ "$words[i]" == --tag || "$words[i]" == -t ]]; then
      tag="$words[i+1]"
      break
    fi
    if [[ "$words[i]" == --tag=* ]]; then
      tag="${words[i]#--tag=}"
      break
    fi
  done

  local cache_file="$HOME/.cache/oc-sandbox/profiles-${tag}"
  if [[ -f "$cache_file" ]]; then
    _values 'profiles' $(cat "$cache_file")
  fi
  _oc-sandbox_maybe_refresh_cache
}

_oc-sandbox_shells() {
  _values 'shells' zsh
}

_oc-sandbox_maybe_refresh_cache() {
  local meta_file="$HOME/.cache/oc-sandbox/cache.meta"
  local images_ts=0
  if [[ -f "$meta_file" ]]; then
    images_ts=$(grep '^images_timestamp=' "$meta_file" 2>/dev/null | cut -d= -f2)
  fi
  local now=$(( $(date +%s) ))
  local max_age=$(( 24 * 60 * 60 ))
  if (( (now - images_ts) > max_age )); then
    (oc-sandbox completion --refresh >/dev/null 2>&1 &)
  fi
}

_oc-sandbox "$@"
```

### Key Design Decisions in the Completion Function

1. **`_values` for dynamic completions** — Uses Zsh's `_values` for image tags and profiles, providing a clean list UI
2. **Context-aware profile completion** — `_oc-sandbox_profiles` scans the command line for `--tag`/`-t` to determine which profiles file to read (default: `main`)
3. **`--tag=*` format handled** — Supports both `--tag main` and `--tag=main` argument styles
4. **Background refresh** — `_oc-sandbox_maybe_refresh_cache` checks the meta file timestamp and spawns a background refresh if stale
5. **`_files -/`** — The PATH argument for `run` completes directories only
6. **Works both when sourced and in `.zshrc`** — The function is self-contained and registers via `_oc-sandbox "$@"` at the bottom

**New `setup_completions` function:**

```bash
setup_completions() {
  local shell="$(basename "${SHELL:-}")"
  case "$shell" in
    zsh)
      local rc_file="${HOME}/.zshrc"
      local completion_line='source <(oc-sandbox completion zsh)'

      # Handle symlinked .zshrc
      if [ -L "$rc_file" ]; then
        info ".zshrc is a symlink; completions will be added to the target file"
      fi

      # Create .zshrc if it doesn't exist
      if [ ! -f "$rc_file" ]; then
        touch "$rc_file"
      fi

      # Check if completion line already exists (idempotent)
      if grep -qF "$completion_line" "$rc_file" 2>/dev/null; then
        info "Shell completions already set up in ${rc_file}"
      else
        echo '' >> "$rc_file"
        echo "$completion_line" >> "$rc_file"
        info "Added shell completions to ${rc_file}"
      fi
      ;;
    bash)
      warn "Bash completion is not yet implemented. See 'oc-sandbox completion --help' for available shells."
      ;;
    fish)
      warn "Fish completion is not yet implemented. See 'oc-sandbox completion --help' for available shells."
      ;;
    *)
      warn "Unknown shell '${shell}'. Add to your shell config: source <(oc-sandbox completion zsh)"
      ;;
  esac

  # Seed the completion cache
  oc-sandbox completion --refresh >/dev/null 2>&1 || true
}
```

### `oc-sandbox uninstall` Changes

Add completion cleanup and cache removal after symlink removal.

**New `cleanup_completions` function:**

```bash
cleanup_completions() {
  local shell="$(basename "${SHELL:-}")"
  case "$shell" in
    zsh)
      local rc_file="${HOME}/.zshrc"
      local completion_line='source <(oc-sandbox completion zsh)'

      if [ -f "$rc_file" ]; then
        if [ -L "$rc_file" ]; then
          info ".zshrc is a symlink; removing completions from the target file"
        fi

        # Remove the completion line
        local tmp_file
        tmp_file=$(mktemp)
        grep -vF "$completion_line" "$rc_file" > "$tmp_file" || true
        mv "$tmp_file" "$rc_file"
        info "Removed shell completions from ${rc_file}"
      fi
      ;;
    # bash/fish: nothing to clean up (not implemented yet)
  esac
}
```

```bash
cleanup_cache() {
  local cache_dir="${HOME}/.cache/oc-sandbox"
  if [ -d "$cache_dir" ]; then
    rm -rf "$cache_dir"
    info "Removed completion cache from ${cache_dir}"
  fi
}
```

## Symlink Handling

When `.zshrc` is a symlink (common in dotfile-managed setups), all operations work correctly because:

- `echo >> ~/.zshrc` follows the symlink and writes to the target file
- `grep` follows the symlink and reads from the target file
- `touch` follows the symlink when creating the file

The implementation includes an informational message when `.zshrc` is detected as a symlink, so users know their actual config file is being modified.

## New `cmd_completion` Function

```bash
cmd_completion() {
  local refresh="false"
  local tag=""
  local shell_arg=""

  while [[ $# -gt 0 ]]; do
    case "$1" in
    --refresh)
      refresh="true"
      shift
      ;;
    --tag)
      if [ -z "${2:-}" ]; then
        error "Option --tag requires a value."
      fi
      tag="$2"
      shift 2
      ;;
    -h | --help)
      usage_completion
      exit 0
      ;;
    -*)
      error "Unknown option: $1\nUse 'oc-sandbox completion --help' for usage information."
      ;;
    *)
      # Shell argument
      if [ -n "$shell_arg" ]; then
        error "Unknown argument: $1\nUse 'oc-sandbox completion --help' for usage information."
      fi
      shell_arg="$1"
      shift
      ;;
    esac
  done

  if [ "$refresh" = "true" ]; then
    cmd_completion_refresh "$tag"
    return
  fi

  if [ -z "${shell_arg:-}" ]; then
    error "Shell name required. Use 'oc-sandbox completion --help' for available shells."
  fi

  case "$shell_arg" in
    zsh)
      cmd_completion_zsh
      ;;
    bash)
      echo "Bash completion is not yet implemented. See 'oc-sandbox completion --help' for available shells."
      ;;
    fish)
      echo "Fish completion is not yet implemented. See 'oc-sandbox completion --help' for available shells."
      ;;
    *)
      error "Unknown shell '${shell_arg}'. Available shells: zsh"
      ;;
  esac
}
```

### `cmd_completion_refresh` Function

```bash
cmd_completion_refresh() {
  local specific_tag="${1:-}"

  # Create cache directory if needed
  local cache_dir="${HOME}/.cache/oc-sandbox"
  mkdir -p "$cache_dir"

  # Check if podman is available
  if ! command -v podman &>/dev/null; then
    warn "podman not found. Cache refresh skipped."
    return 0
  fi

  local now
  now=$(date +%s)

  # Refresh images list
  local images
  images=$(podman images --format '{{.Repository}}:{{.Tag}}' 2>/dev/null | grep "^${IMAGE_BASE}:" | sed "s|^${IMAGE_BASE}:||" || true)
  echo "$images" > "$cache_dir/images"
  echo "images_timestamp=${now}" > "$cache_dir/cache.meta"

  # Refresh profiles for specific tag or all tags
  if [ -n "$specific_tag" ]; then
    _refresh_profiles_for_tag "$specific_tag" "$cache_dir" "$now"
  else
    # Refresh profiles for all known tags
    while IFS= read -r tag_name; do
      [ -z "$tag_name" ] && continue
      _refresh_profiles_for_tag "$tag_name" "$cache_dir" "$now"
    done < "$cache_dir/images"
  fi
}

_refresh_profiles_for_tag() {
  local tag_name="$1"
  local cache_dir="$2"
  local now="$3"
  local image_name="${IMAGE_BASE}:${tag_name}"

  if podman image exists "$image_name" 2>/dev/null; then
    local profiles
    profiles=$(podman run --rm "$image_name" ls -1 /home/sandbox/opencode-config/profiles/ 2>/dev/null | grep -v '^\.' || true)
    echo "$profiles" > "$cache_dir/profiles-${tag_name}"
    echo "profiles-${tag_name}_timestamp=${now}" >> "$cache_dir/cache.meta"
  fi
}
```
